## как использовать 
```
git clone https://github.com/Aukvary/ML-Case.git
cd ML-Case
Docker compose up --build
```
- `http://localhost:8000/docs` - api bus
- `http://localhost:5001/docs` - parser
- `http://localhost:5002/docs` - ai model

## bus api
### лёша
- отвечает за взаимодействие фронта, модели и базы данных

## ai-model
### ваня литвин
- главная задача преобразовать текст в вектор

## parser
### кому не лень
- нужно преобразовать файлы(pdf-text, pdf-image, docx) в текст

## data base
### лёша
- хранить данные в виде векторов для поиска

## front
### ваня томкович
- загрузка файлов
  - возможность добавлять теги приватности при добавлении 
- поиск файлов
- админ панель для добавления

#### добавление файла
- админ/обычный чел кидает файл/файлы/zip с файлами
- фронт говорит, кто кинул(админ или нет) и данные
- апишка в зависимости от отправителя игнорит или кидает все файлы в парсер
- полученные данные закидываются в модельку, которая превращает их в вектор
- вектор закидывается в базу данных
- на фронте показывается, что всё добавлено/нет прав

#### поиск файла
- чел вбивает список слов/какой-то запрос
- фронт кидает запрос апишке
- апишка кидает слова в модельку, чтобы превратить в вектор
- апишка обращается к базе данных, чтобы получить n документов, которые большье всего подходят

#### управление файлами
- если ты админ, то у файлов можно сделать доп кнопки типа удлалить, изменить
- при удалении просто удаляем из базы данных
- при изменении повторяем действия из добавления
